# GENAI_PROJECT
This project aims to simulate a real life interview between human and AI. Current technical interview processes often suffer from static questioning that fails to accurately gauge a candidate's depth of knowledge or adapt to their specific strengths. Through this project, I aim to introduce an AI-driven adaptive interviewing system utilizing the Gemini 3.1 Flash Lite Preview model. My approach employs a dynamic "Knowledge Map" architecture that allows the model to adjust its difficulty levels and topic focus in real-time based on candidate performance. By treating the interview as a function space optimization problem, the system minimizes assessment bias and maximizes information gain regarding a candidate's technical proficiency. The results demonstrate that this adaptive mechanism provides a more nuanced evaluation compared to traditional fixed-script methodologies. 
The objective of this project is to move beyond the "one-size-fits-all" approach to technical evaluations. I leverage Large Language Models (LLMs) to create a conversational interface that can pivot between various technical domains: such as machine learning, data structures, and system design, based on the interviewee's previous answers. This project explores the balance between aleatoric uncertainty (noise in candidate communication) and epistemic uncertainty (the system’s lack of knowledge about the candidate’s true skill level). 
The motivation for this project stemmed from the frustration of repetitive, non-adaptive technical assessments. Our goal was to build a tool that feels like a conversation with a senior engineer, one that probes deeper when you show strength and pivots when it identifies a gap. We focused on bridging the gap between a standard Python script and a generative AI engine to create a seamless user experience using Streamlit.
